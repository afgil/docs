---
title: "Desenrolar Entidad"
openapi: "DELETE /credentials/{master_entity_id}"
---

## Que hace

- Elimina la asociación entre el usuario de la api key y la entidad maestra
- No elimina la entidad maestra ni sus credenciales, solo desvincula al usuario
- Otros usuarios que estén asociados a la misma entidad mantienen su acceso

## Ejemplos de uso

- Dejar de administrar una entidad desde la api key actual
- Revertir un enrolamiento hecho por error
- Limpiar entidades que ya no corresponden a la integración

### Obtener el master_entity_id

El `master_entity_id` es el ID unico de la empresa en Tupana que se esta desasociando. Para obtener este ID, debes usar el endpoint `/master-entities`:

1. **Buscar la entidad por RUT** usando `GET /master-entities?rut=76543210-1`
2. **Obtener el ID** de la respuesta (campo `id`)
3. **Usar ese ID** como `master_entity_id` en la ruta del endpoint

### Importante

- El usuario de la api key debe estar asociado a la entidad, de lo contrario se retorna `404`
- Esta acción no puede deshacerse automáticamente: para volver a acceder a la entidad se debe crear una nueva credencial (ver [Crear Credencial](/api-reference/credentials/create))
