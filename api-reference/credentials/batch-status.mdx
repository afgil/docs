---
title: "Consultar Lote de Credenciales"
openapi: "GET /credentials/batch/{batch_id}"
---

## Que hace

- Devuelve el estado actual de un batch creado con [POST /credentials/batch](/api-reference/credentials/batch) y el estado individual de cada credencial
- Sirve como respaldo al webhook (`credential.validated` / `credential.invalid`) si no esta configurado, o para hacer polling mientras el batch termina de procesarse

## Estados del batch

| `status` | Significado |
|----------|-------------|
| `processing` | Aun quedan credenciales por resolver |
| `completed` | Todas las credenciales quedaron validas |
| `partial` | Algunas credenciales quedaron validas y otras no |
| `failed` | Ninguna credencial quedo valida |

## Estados de cada item

| `status` del item | Significado |
|--------------------|-------------|
| `pending` | Aun no se ha procesado |
| `processing` | Se esta validando contra el SII en este momento |
| `valid` | Login al SII exitoso |
| `invalid` | Clave incorrecta o cuenta bloqueada (ver `errors.blocked`) |
| `temporary_error` | Error transitorio (red, mantenimiento SII); se reintentara automaticamente |
| `permanent_error` | Se agotaron los reintentos, o la entidad/formato no se pudo resolver |
