---
title: 'Países de exportación'
description: 'Códigos de país del SII aceptados en facturas de exportación (DTE 110, 111 y 112)'
---

Las facturas de exportación (DTE 110), notas de débito (111) y notas de crédito (112) declaran el país de destino de la operación mediante el campo `destination_country_code` dentro de `export_data`.

<Warning>
  El `destination_country_code` **no es el código ISO 3166-1** (`US`, `BR`, `AR`). Es el **código de país del SII** (tabla de Aduana), enviado como *string*. Por ejemplo, Estados Unidos es `"225"`, no `"US"`.
</Warning>

## Obtener la lista de países

El listado completo (código y nombre) está disponible en un endpoint público, sin autenticación:

```bash
curl https://api.tupana.ai/api/documents/export-countries/
```

```json Respuesta
[
  { "code": "224", "name": "ARGENTINA" },
  { "code": "220", "name": "BRASIL" },
  { "code": "225", "name": "U.S.A." }
]
```

<Info>
  Este endpoint es la **fuente de verdad**. La tabla de abajo es una copia de referencia; ante cualquier diferencia, prevalece la respuesta del endpoint.
</Info>

## Uso en `export_data`

<ParamField body="destination_country_code" type="string" required>
  Código de país del SII del destino final de la exportación (ver tabla). Se envía como texto, por ejemplo `"225"`.
</ParamField>

```json export_data
{
  "export_data": {
    "destination_country_code": "225"
  }
}
```

<Note>
  El `destination_country_code` debe coincidir con el país del receptor extranjero (`document_receiver.country`).
</Note>
